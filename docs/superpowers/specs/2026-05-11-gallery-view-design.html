A per-pane grid view for fman — thumbnails for images, file-type icons for the rest, with metadata laid over the corners and a keyboard you can lean on.

01
Intent

What this feature is for — and, more importantly, what it is not for.

Goals & non-goals.

What it does

  • Adds an alternative file view (gallery / icon grid) selectable per pane.
  • Shows image thumbnails for image files, file-type icons elsewhere.
  • Overlays key metadata on image tiles — extension, resolution, size.
  • Leaves the existing list view untouched and fully working.
  • Persists mode and tile size per pane across sessions.

What it doesn't do

  • Replace or rewrite the existing FileListView.
  • Recurse into folders for nested previews.
  • Generate video, document, or PDF thumbnails.
  • Mouse drag-resize — keyboard Ctrl/⌘ + +/− only.
02
Architecture

Two views share a model and a selection. The pane just shows one at a time.

Architecture.

Both views bind to the same SortedFileSystemModel and share a single QItemSelectionModel, so cursor and selection stay coherent across toggles. The pane is the one place that knows which view is on screen — it swaps them inside a QStackedWidget and exposes two small methods to the rest of the app.

DirectoryPaneWidget
LocationBar
QStackedWidgetnew
FileListView — existing
GalleryView — new
Both children · same model · same selection model

The pane gains a public surface that the rest of fman can reach for:

  • set_view_mode(mode: 'list' | 'gallery') — switch the active view.
  • get_view_mode() -> 'list' | 'gallery' — read the current mode.

Both values — mode and tile size — persist per pane in fman's session state, keyed to the pane identity.

03
Components

Three new pieces. Each has one job.

Components in three movements.

I. Viewsrc/main/python/fman/impl/view/gallery.py

GalleryView

Subclass of QListView configured for icon-mode rendering. Reuses the existing DragAndDrop mixin so drop targets, drag previews, and ghost cursors all match list-view behaviour.

  • viewMode = IconMode, movement = Static, resizeMode = Adjust.
  • wordWrap = True, uniformItemSizes = True, flow = LeftToRight.
  • Owns its own GalleryItemDelegate for tile rendering.
  • Catches Ctrl/⌘ + + / − / 0 in keyPressEvent.
  • Emits the same public signals (doubleClicked, context-menu events) the pane already listens for.

III. Cachesrc/main/python/fman/impl/view/thumbnails.py

ThumbnailCache

Persistent disk cache under fman's user cache directory at <cache_dir>/thumbnails/. Cache keys are SHA1(absolute_path | mtime_ns | size_bucket) — file changes invalidate automatically.

  • Size buckets: 128, 256, 512 px. Tiles downscale the nearest-larger bucket.
  • get(url, size_bucket) — sync cache lookup, returns QPixmap | None.
  • request(url, size_bucket, callback) — schedules async generation on a QThreadPool, callback fires on the main thread.
  • Resolution metadata is read once via QImageReader.size() and held in an in-memory WeakValueDictionary keyed by (path, mtime).
  • Supported formats: QImageReader.supportedImageFormats().
04
Filename rule

First three are sacred. Last five if there's room. Don't truncate when you don't need to.

The truncation rule.

First three always. Last five when we can. Never cut what doesn't need cutting.

  1. Strip the extension first.
  2. If the stripped name fits the tile width → show it whole. No truncation when avoidable.
  3. Otherwise, try <first3>…<last5> — if it fits, that's the result.
  4. If even that's too wide → drop the suffix and show <first3>…. The first 3 always wins.

05
Tile size

Continuous resize, ±20 px. Thumbnail grows with the tile.

Resizing tiles.

Keyboard-only. Continuous steps of ±20 px per press. The icon and the grid grow in lockstep — set both iconSize and gridSize on each step so the thumbnail and label stay in proportion.

Ctrl/⌘ + grow  ·  Ctrl/⌘ − shrink  ·  Ctrl/⌘ 0 reset to default.

Tile size persists per pane in session state.

06
Command

Keyboard-driven, palette-discoverable. The fman way.

View toggle.

A new application command, toggle_gallery_view, registered in the core plugin registry.

  • Default binding: Ctrl + G  /  ⌘ + G.
  • Palette caption: “Toggle Gallery View.”
  • Flips the focused pane's view mode between 'list' and 'gallery'.
07
Navigation

Same pane API; different keys underneath.

Keyboard navigation.

QListView's IconMode already handles 2D arrow navigation, Home/End, and PageUp/PageDown out of the box. DirectoryPaneWidget's existing move_cursor_* methods need parallel wiring on the gallery view so the public pane API stays uniform — they delegate to QListView's built-ins instead of the custom QTableView logic.

09
Testing

Unit for rules. Integration for flow.

Testing plan.

Unit · src/unittest/python/fman_unittest/impl/view/

  • test_gallery_filename_truncation.py — short, exact-fit, just-over, very long, names shorter than 3, first-3 always preserved, last-5 dropped when tile too narrow.
  • test_gallery_overlay_layout.py — overlay switches to stacked at tile_width < 140 px; badges never overlap each other or the right edge.
  • test_thumbnail_cache.py — cache key collisions, mtime invalidation, size-bucket selection, eviction.

Integration · src/integrationtest/python/

  • Toggle modes via command, verify cursor + selection preserved.
  • Ctrl++ / Ctrl+− step, clamping at min/max, Ctrl+0 reset.
  • Image file shows overlays; non-image file does not.
  • Persistence: set gallery + tile 240 px, restart, verify restored.
